Le Bridge (ancien)

  • Asynchrone : chaque appel JS → Native passe par une file d'attente
  • Serialisation JSON : les donnees sont converties en JSON string, envoyees, puis parsees cote natif
  • Goulot d'etranglement : animations, listes defilantes et gestes souffrent du delai
  • Pas de references partagees : JS et Native ne peuvent pas acceder aux memes objets

Le JSI (nouveau)

  • Synchrone : appels directs sans file d'attente
  • Pas de serialisation : les objets C++ sont accessibles directement depuis JS
  • References partagees : JS peut tenir une reference a un objet natif (HostObject)
  • Turbo Modules : chargement paresseux, typage compile (Codegen)
